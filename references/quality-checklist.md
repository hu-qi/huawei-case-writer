# Quality checklist for Huawei case articles

Review the article against this checklist before returning it.

## Structural compliance

- The five top-level sections are present.
- Secondary sections are named clearly and consistently.
- The article reads like a runnable case, not a product brochure.
- The section granularity matches the actual operational steps.

## Style compliance with reference cases

- The opening includes a concrete real-world scenario when appropriate.
- `案例流程` includes both a short explanation and a numbered step list.
- Skill-based cases explain the skill directory or important files when useful.
- Prompt-driven cases include at least one realistic prompt example.
- Major steps include explicit verification sentences.

## Operability

- Each core step has an action, key parameter, and expected verification point.
- Important dependencies and prerequisites are stated before use.
- Commands or prompts can be copied directly with minimal edits.
- Version-sensitive dependencies are pinned where possible.

## Publishing quality

- The opening paragraph is short and compelling.
- Audience and duration fit the actual complexity.
- Resource table is realistic and avoids unsupported claims.
- Risky unknowns are marked as `待补充` or `以实际为准`.
- No internal editorial notes remain.

## Cost and cleanup

- Paid services are disclosed.
- Cleanup or release instructions exist for billable resources.
- The wording reminds readers to release resources promptly.

## Common fixes

- Replace vague verbs like `进行配置` with concrete actions.
- Replace long prose with step bullets or subsections.
- Move oversized code blocks to only the key fragments.
- Add a verification sentence after each major operation.
- If the case is about AI dialogue or skill invocation, add the actual prompt text.
